Now that people have started to embrace coding agents instead of looking down on those who use them to ship software like they did back in July 2025 (some still do but now they are just in a minority) they come to the same people asking what sub to get (atleast in my office).

For the most part when these conversations were happening I hadn’t used codex myself because claude code had good rate limits and I was getting the hang of shipping with claude so was not experimenting much but right around the that time [Amp](https://ampcode.com) launched their [deep mode](https://ampcode.com/news/deep-mode) powered by GPT5.2-Codex. I have been a long standing fan of Amp, in my opinion it is the best agentic tool out there in the market right now, it's just that it is expensive to use since the amp team cannot actually provide us with subsidies like Anthropic and OpenAI (which even they are working towards rescinding now) which results in a much smaller footfall as compared to ClaudeCode or Codex atleast in the Indian ecosystem per my opinion.

The amp team released making a comparison that Opus is a model eager to do work and is more like a doer model whereas Codex is a model which likes to go deep (hence the name) and just read code and figure out plans of action and then making small surgical edits. Now when I heard this and tried the deep mode myself, I realised even though these are products from two arch rivals (so much so that their CEOs can't even touch each other - [ref](https://youtube.com/shorts/vS68BsFWIqY?si=uLSvCnqwRFJlJNqY)) these models are actually very complimentary and ever since then I started using Deep mode to help in planning the task grounding Opus's over the top output in planning (due to it's eagerness and the way I use it) and helps me remove the code duplication suggestions and keep the design sane, so, the simple workflow becomes, jam with claude code on a feature, scope it out get it to spit out a design doc, feed that doc in Amp deep mode get it grounded turn it into an implementation plan and get the verification criteria sorted for each phase and then give that document back to claude code to implement and once it is done implementing, ask the thread in the deep mode to go and verify the implementation and fix the bugs claude has created and this has grounded the implementation a lot to the point where a fairly complex system involving firecracker VMs and remote agent orchestration has been vibe coded completely without me ever reading code and it always and always works reliably for me on my tail scale network, simply because these models are pretty good at complimenting the short comings of each other, which makes me think it's not Opus or Codex but rather Opus *and* Codex.
